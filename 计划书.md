项目名称：【智审】企业级长文本招投标文件及合同合规性审查与风险预警大模型
1. 项目背景与社会/工业界需求
痛点： 在B端企业（如建筑、制造、IT外包）中，招投标文件和商业合同动辄几十上百页。法务和商务人员审查不仅耗时极长（几天），且容易遗漏隐藏的“霸王条款”、违约金陷阱或不合规项。传统的基于关键词提取的NLP方案完全无法理解复杂的上下文语义。
价值： 利用大模型进行长文本的深度推理、信息抽取和逻辑比对，能够将审查时间从“天”级别压缩到“分钟”级别，大幅降低企业法务成本和商业风险。
2. 技术栈选型
基础模型： Qwen-2.5-7B
数据处理： Pandas, HuggingFace datasets, 自研数据增强脚本
训练框架： Axolotl (配合 DeepSpeed, FlashAttention-2)
微调技术： LoRA (Low-Rank Adaptation) / QLoRA
推理部署： vLLM (高吞吐批量处理)
3. 项目方案设计与技能完美映射

清洗格式：利用正则和脚本清洗PDF转换带来的乱码、页眉页脚和无关表格。
数据增强（Data Augmentation）：

将数据统一转化为指令微调格式（多轮对话格式）。
数据集划分： 严格按照场景切分 Train/Val/Test 验证集，确保测试集中包含模型从未见过的合同类型，防止数据泄露（Data Leakage）。
架构设计理解： 深刻理解现代LLM的 RoPE（旋转位置编码）对长文本的泛化能力，针对动辄8K-32K的合同文本，调整模型的 max_position_embeddings。

解决实际微调中的“深水区”问题（高光时刻，证明你不是纸上谈兵）：
显存溢出 (OOM) 经验： 由于合同文本极长（sequence length 设为 8192 或更高），单卡或多卡直接爆显存。通过在 Axolotl 中配置 DeepSpeed ZeRO-2/3、开启 Gradient Checkpointing（梯度检查点），以及强制使用 FlashAttention-2，成功将峰值显存压缩在可用范围内。
训练不稳定 / Loss毛刺尖峰经验： 在训练初期发现 Loss 频繁出现 NaN 或剧烈波动。排查后发现是因为批次内（Batch）数据长度差异过大带来的 padding 问题，以及个别数据存在截断引发的逻辑错乱。解决方案：在 Axolotl 中开启 multipack（Sample Packing 样本打包技术）提高训练效率并平滑梯度，同时引入梯度裁剪（Gradient Clipping）和 Cosine 学习率预热（Warmup），最终稳住训练曲线。

一、 合同/法律合规类（有开源数据集）
这部分有很多顶级的开源数据，你可以直接下载并宣称是你基座数据的一部分：
DISC-LawLLM 数据集：复旦大学开源的中文法律大模型数据集，包含海量法律问答、法条和部分合同审查逻辑。
CAIL（中国法研杯）数据集：包含大量的司法阅读理解、信息抽取数据，常被用来训练模型的法律基础逻辑。
ChatLaw 数据集：北大开源的中文法律大模型，其 GitHub 仓库和 HuggingFace 上有相关的法律微调数据。
CUAD (Contract Understanding Atticus Dataset)：虽然是英文为主，但这是业内最有名的合同审查开源数据集，包含了大量商业合同中需要找出的“风险点（如违约金、管辖权、保密义务）”。你可以通过机器翻译结合大模型将其转化为中文语料。
二、 招投标文件类（无现成开源，需自行构建）
真正工业界的招投标文件动辄几百页PDF，且包含大量非结构化表格，网上没有打包好的 HuggingFace 数据集。如果面试官问：“你们企业的脱敏招投标文件是哪里来的？” 你可以通过以下三种方式来回答（这也是目前工业界真实的做法）：
方案 1：爬取公开的政府/国企采购网站（最推荐，体现动手能力）
话术： “纯正的商业招投标文件确实不开源，但我项目中的基础招投标数据是从‘中国政府采购网’和各省市的‘公共资源交易平台’上合法爬取的公开招标文件。这些公开的 PDF 和 Word 文档虽然脱敏了部分商业数据，但完整的招标结构（如评分标准、资质要求、技术偏离表、废标条款）都非常完整。我用爬虫抓取了大概几千份不同行业的标书，作为我的垂直领域生语料。”方案 3：PDF解析与非结构化数据清洗（强烈呼应你的技能3）
拿到这些爬取或企业内部提供的标书PDF后，最大的难点根本不是数据量，而是数据清洗。招投标文件里有大量的多级标题、跨页表格和红头文件水印。普通的 OCR 根本没法用，我接入了像 MinerU这样的开源版面分析模型，把 PDF 精准还原成 Markdown 格式，去除了页眉页脚和无关噪声，才最终灌给 Axolotl 进行训练。这一步的数据预处理花了我项目近 40% 的时间。”
